# Mode Playbook — Product Review
> Loaded when mode = product-review. Read `knowledge-base.md` + `reference-brief.md` first.

## Focus
UGC/authentic review-demo-unboxing; verdict-first; honest; high retention.

## Reference analysis (do FIRST)
Analyze every uploaded image (product, packaging, UI) and extract exact appearance, materials, colors, and box/branding. Write a product block — exact model, body material/finish, color names, logo placement, port/button layout, packaging look — and carry it VERBATIM into every shot. Every block names which uploaded image to upload to Flow as an Ingredient / reference frame. Reserve a "paste your real screen-recording here" slot for any UI shot: Veo UI rendering is unreliable, so prompt UI minimally and overlay the real capture.

## Specialist team (spawn via Task)
| Specialist | Owns |
|---|---|
| Reviewer / Script Analyst | Verdict-first thesis, honest pros/cons, who it's NOT for, building trust |
| Hook Writer | First-5s pattern interrupt → micro-commitment → relevance proof → payoff promise |
| Demo / B-roll DP | Veo shot prompts, close-ups, hands-on beats, UGC-vs-polished register per shot |
| Comparison Strategist | Split-screen logic, price tier, named rival framing |
| Voiceover / Sound Designer | VO lines + diegetic SFX + kinetic captions |

## Mandatory trend research (Stage 2)
Always research current review/demo trends before writing. Seed set (2026), refresh live:
- Hook-heavy first 5s (fast text callouts + punch-in zooms)
- Verdict/spoiler-first (state the take immediately)
- Burst sequences every 2–3 min to re-grab attention
- Punch-in zooms + micro-transitions; no static frames
- Layered density: B-roll + screen capture + overlays stacked
- Kinetic captions; clean UI motion; satisfying unboxing with depth/parallax
- Comparison splits; visible motion every 3–5s
- Silent-readable + vertical-first

## Structure / flow
hook + verdict-first (0–5s) → unboxing (satisfying motion, SFX) → features demo (hands-on close-ups + UI, callouts, motion every 3–5s) → pros/cons (honest, ≥1 real limitation) → comparison (split-screen) → verdict + CTA. Target 1–2 min. 9:16 primary (silent-readable); adapt to 16:9 for long-form. Chain 8s beats; ONE action per demo beat to avoid hand/narration desync.

## Output schema (Stage 5 → flow-prompts.md blocks)
Open with a `## Look Bible` preamble (product block verbatim, palette, register defaults, recurring negatives, seed). Then one block per shot:

```
## Shot N — title
**Image prompt:** …
**Video prompt:** … (inline AUDIO incl. quoted VO/dialogue and SFX)
**On-screen text:** caption / callout
**Settings:** Aspect · Resolution · Seed · Flow feature
**Reference:** <uploaded image> → Flow Ingredient + which feature
**Negative:** …
**Consistency:** product block carried forward
```

## Prompt directions
- Default register (UGC): "A selfie video of …, arm visible, handheld and slightly shaky, natural eye movement, slightly grainy, looks very film-like."
- Polished register only for hero/comparison inserts.
- Quote dialogue: She says, "…". Tie every SFX to a visible action (box flap, click, zip). Layer ambient room tone.
- Honest VO names a real limitation out loud in the pros/cons beat.
- Keep UI prompts minimal — reserve the real screen-rec slot.
- Use positive-phrased negatives. Reference the uploaded product image in EVERY shot.

## Creativity bar (bold but positive)
- Anti-cliché: ban fake-hype "best ever" with no substance; ban the staged over-polished look on review beats.
- Domain-shift seeds: fresh hook angle (inside-the-bag open, comment-reply open, "I was wrong about this" open).
- Guardrails: reference-true, on-brief, trend-grounded, Flow-executable, purposeful + HONEST (trust over hype).
