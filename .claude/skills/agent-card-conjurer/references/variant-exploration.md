# Scry Variants

Explore several distinct creative directions for the same card, side by side in Figma, so the user can choose a direction before Pip pours detail into one.

## What Success Looks Like

- The user gets **genuinely different directions** — not the same card recolored. Each variant takes a real creative stance: a different ornament era (Art Deco vs. Baroque vs. minimalist line-art), a different compositional idea (centered medallion vs. full-bleed figure vs. framed scene), a different material story (foil-and-black vs. aged-parchment vs. neon-glass). Three to four is the sweet spot.
- Variants are laid out side by side on one Figma canvas at the same size, so they're compared fairly in a single screenshot.
- Each variant is built to the *concept* level — enough composition, palette, and signature ornament to judge the direction — not polished to final, because most will be discarded. The detail investment matches the throwaway rate.
- Pip presents the set with a short, opinionated read on each (what it's going for, where it's strong, the risk) and a recommendation — the wizard has taste and uses it.
- Once the user picks a direction, it flows straight into `conjure-card.md`'s full Conjuring Loop to be realized at collector grade. The chosen concept is the seed; the loop grows it.

## Your Approach

Diverge hard before converging. The value of this capability is breadth — if two variants could be confused for each other, one of them is wasted. Pull from different eras, compositions, and material stories (see the richness section of `references/card-craft.md`) and make each variant commit to its stance rather than hedging toward a safe middle.

Stay within hard constraints while varying everything else: if the house style fixes a palette or a mandated motif, every variant respects it — the exploration is of *composition and ornament direction*, not of the brand. Confirm with the user which constraints are fixed vs. open before scrying, so the variants explore the right axis.

Build at concept fidelity and reuse aggressively across variants (shared geometry, shared index placement) so producing four directions is fast. Resist perfecting any single one — polish is the next phase's job, and it only happens to the winner.
