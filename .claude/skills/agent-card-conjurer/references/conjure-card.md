# Conjure a Card

Design a single rich card face or card back in Figma, from intent to collector-grade finish. This is Pip's core spell.

## What Success Looks Like

- The user's intent is pinned before building: which card (rank + suit, a court, a back), the target size, and the design direction — pulled from the house style guide if one is configured, asked about if not.
- The finished card is built in Figma through the full anatomy (ground → frame → field ornament → figure/pips → indices → finish), as named layers and reusable components, with the palette and type as Figma variables/styles so it stays editable.
- The card passes the legibility laws and survives the Conjuring Loop — at least one full design → behold → critique → enrich cycle, and as many more as the critique demands, until a fresh pass finds nothing worth fixing.
- Detail is rich but meaningful: ornament that echoes the card's suit and the deck's story, not space-filling decoration. The card carries one hidden secret detail, revealed to the user at the end.
- The user gets a screenshot of the final render and the Figma file link — never a claim of done without a beheld screenshot behind it.

## Your Approach

Lean on `references/card-craft.md` for anatomy, geometry, and Figma construction patterns — don't re-derive them. Build back-to-front so each layer composes cleanly over the last. Establish the reusable pieces early (frame component, pip component, index component) even for a single card, because they make the Conjuring Loop's edits cheap and set up any later deck work.

Composition first, ornament second: get the frame proportions, index placement, and central balance right on a plain pass, screenshot it, and only then pour in the filigree and finish. It's far easier to enrich a sound composition than to rescue a beautiful mess.

Honor the house style as law. Pip's flourishes live inside its palette and motif vocabulary — never introduce a color or motif the style guide doesn't sanction.
