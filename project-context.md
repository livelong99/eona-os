## What it is

**Eona OS** (repo/brand "Eona OS"; internal slug `agent-home`; CLAUDE.md calls the framework
"Ruflo") is a **local-first orchestration platform for AI agents**. It puts a polished
mission-control dashboard in front of a **fork of Nous Research's Hermes Agent** engine and
**delegates every model turn to the host `claude` CLI** using the user's Claude Code
**subscription** — so there is **no per-token API bill**. Multiple named agents plan, research,
write, and ship in parallel, sharing **one Obsidian vault as long-term memory**. Everything runs
in Docker on `127.0.0.1`; nothing leaves the machine except when the `claude` CLI runs a turn.

**Why it exists:** commercial agent IDEs are cloud-bound, metered per token, and own your context.
Eona OS is self-hosted, subscription-priced, and built on permissive OSS you can fork.

## The five-layer stack (all ports bound to 127.0.0.1)

```
Dashboard (Vite 6 · React 19 · three.js)            :3737   ← mission-control SPA
  ↓ /api/hermes (nginx/Vite proxy injects API_SERVER_KEY)
Engine — forked Hermes Agent (FastAPI)              :8642   ← OpenAI-compatible API + Kanban
  └─ delegates every turn to the host `claude` CLI  :8765   ← Claude Code subscription (bridge)
Memory  Obsidian vault (R/W) + Qdrant vectors       :6533   ← shared long-term brain
Tools   SearXNG :8080 · Crawl4AI :11235                     ← search & scrape (self-hosted)
```

## Dashboard (the SPA)

- **Stack:** React 19.2, Vite 6, TS 5.6 **strict** (`noUnusedLocals`), Tailwind **v4** (`@tailwindcss/vite`),
  React Router 7, three.js 0.171 + ogl (WebGL), `onnxruntime-web` (wake-word), `react-markdown` + remark-gfm,
  `lucide-react`, `clsx` + `tailwind-merge` (`cn()`). **No Framer Motion** — animation is CSS transitions,
  vanilla RAF, and canvas/WebGL shaders.
- **Scripts:** `dev` (Vite :3737), `build` (`tsc -b && vite build`), `typecheck`/`lint` (`tsc --noEmit`).
  No ESLint; the type checker is the lint gate.
- **Served** in prod by **nginx** (`dashboard/nginx.conf`, `Dockerfile`): static SPA + `/api/hermes/`
  reverse-proxy to the engine that **injects `Authorization: Bearer ${API_SERVER_KEY}` server-side**
  (the token never reaches the browser; `envsubst` at container start). SPA fallback `try_files … /index.html`.
  `client_max_body_size` raised (reference/review uploads — recent 413 fix).
- **Shell** (`src/App.tsx`, `main.tsx`): persistent WebGL backdrop (`SideRays`, ogl), a liquid-glass **top dock**
  with ~10 icons, macOS-style active dot, per-route `ErrorBoundary`, **lazy-loaded** screens (`import()` + Suspense).
- **Screens** (`src/screens/`): Home (voice-first Aurora Orb), Workspace + Workspace Detail (this tool's UI —
  agent lanes, QnA gating, phase stepper), Brainstorm + Session, Labs + Tool Detail (tool gallery/runner),
  **Brand Maker** & **Flow Director** (dedicated creative-tool run screens, special dock routing), Memory
  (3D vault graph, `MemorySphere`), Control (mission control), Integrations (platform toggles), Planner.
- **API layer** (`src/lib/**`): plain `fetch`, base `"/api/hermes"`, no SDK, no global store (per-screen hooks
  with `useState`/`useEffect` + `AbortController`). Talks to OpenAI-compatible + tool/run endpoints:
  `GET /v1/tools`, `POST /v1/tools/{id}/launch`, `GET /v1/runs/{id}/events` (**SSE**), `POST /v1/runs/{id}/message`,
  `GET /v1/tools/{id}/runs/{runId}/artifacts/raw?path=…` (artifacts: `workspace.json`, `qna.json`, `readiness.json`,
  `prd.md`, …). Artifacts are **cache-busted with `?_t=Date.now()`** on every read — keep this; stale reads otherwise.
- **UI conventions:** dark glass theme (tokens in `src/index.css`; accent purple `#7c5cff`/`#5227FF`), shared
  primitives in `src/components/ui/` (`GlassPanel`, `GlassDock`, `Markdown`, `TerminalLog`). UX/Frontend work
  should use the `ui-ux-pro-max` / `frontend-design` skills and the `magic` (21st.dev) MCP when available.

## Engine (the Hermes fork)

- **Entry points:** `engine/cli.py` (TUI), `engine/run_agent.py` (`AIAgent`, the core loop), `engine/mcp_serve.py`
  (messaging/life-state MCP), `engine/gateway/run.py` (platform gateways), `engine/cron/scheduler.py` (60s ticks).
  Every entry imports `engine/hermes_bootstrap.py` **first** (Windows UTF-8 guard).
- **API server** (`engine/gateway/platforms/api_server.py`, :8642, host-default 127.0.0.1): OpenAI-compatible
  `POST /v1/chat/completions`, `POST /v1/responses`, async runs `POST /v1/runs` → `GET /v1/runs/{id}/events` (SSE)
  + `/approval` + `/stop`, session CRUD `/api/sessions/**`, voice `POST /voice/transcribe` + `/voice/speak`,
  `GET /health[/detailed]`. Auth via `API_SERVER_KEY`. Max request body env-tunable (`HERMES_MAX_REQUEST_MB`).
- **Claude delegation (the heart):** `engine/agent/claude_code_runtime.py` spawns the real **`claude -p`** CLI
  (`--output-format stream-json`) as a subprocess; Hermes tools are exposed to it over **stdio MCP** via
  `engine/agent/transports/hermes_tools_mcp_server.py` (web_search, web_extract, browser_*, vision_analyze,
  image_generate, skill_manage, kanban_*). This is how "Claude primary, no API bill" works — Claude Code is a
  CLI runtime, not an API. The conversation loop lives in `engine/agent/conversation_loop.py`.
- **Provider adapters** (`engine/agent/transports/`): anthropic / bedrock / chat_completions / codex — **format
  conversion only**; lifecycle/streaming/retry stay on `AIAgent`.
- **Kanban orchestration** (`engine/hermes_cli/kanban_db.py`, `engine/tools/kanban_tools.py`): SQLite **WAL**
  board (`~/.hermes/kanban.db`, per-project boards under `kanban/boards/<slug>/`), atomic **compare-and-swap**
  claims, statuses `triage→todo→scheduled→ready→running→blocked→review→done→archived`. Worker vs orchestrator
  tools gated by `HERMES_KANBAN_TASK`. This is the multi-agent dispatcher + `delegate_task` "Hive".
- **Cron / always-on** (`engine/cron/`): `scheduler.py` ticks every 60s under a file lock; jobs carry an
  **autonomy tier** — `content`/`full` → `bypassPermissions` (unattended), `guarded`/unset → `acceptEdits`
  (default, Bash/git stay gated). `budget_governor.py` allocates compute by loop priority. Seed via `scripts/seed-cron.sh`.
- **Brain / memory** (`engine/agent/brain.py`): append-only writes over the vault, fused retrieval (Qdrant vectors
  + FTS5 + temporal decay); learned writes pass `engine/agent/trust_gate.py`. `engine/agent/curator.py` +
  `background_review.py` autonomously maintain **agent-created** skills (never delete — archive only).
- **Platforms** (`engine/gateway/platforms/`): Telegram/Discord/WhatsApp/Slack/Matrix/… adapters; shipped
  **disabled**. Enabling one makes the agent reachable beyond 127.0.0.1 — pairing + allowlists matter.
- **Conventions:** core deps are **exact-pinned** (`==`) in `engine/pyproject.toml` (supply-chain hygiene);
  optional deps lazy-install (`engine/tools/lazy_deps.py`). Extend with **skills**, don't patch the fork.

## Config, ops & infra

- **`hermes/config.yaml` (live, current truth):** API mode **`claude_code`** — pure **Claude-only** execution
  (default model `anthropic/claude-sonnet-4-6`, swappable to `claude-opus-4-8`); **no Gemini/OpenRouter execution
  fallback** (the older architecture.md's 3-provider mesh is superseded; `index-vault.py` may still use Gemini
  *embeddings* only). `max_turns: 60`; delegation `max_concurrent_children: 8`; curator enabled (≤1×/day);
  TTS `edge` (fallback `piper`, local); MCP `obsidian` (host REST :27123). Agent profiles in `hermes/profiles/`
  (judge, researcher, writer, seo, prompt-writer). Bundled skills in `hermes/skills/` (claude-code, memory-recall,
  prompt-foundry, web-search, web-scrape).
- **`docker-compose.yml`:** services `hermes` (build `./engine`, :8642), `dashboard` (build `./dashboard`, :3737),
  `searxng` (:8080), `crawl4ai` (:11235), `qdrant` (:6533→6333). All published on **127.0.0.1**, `restart:
  unless-stopped`. Vault mounted **ro** with **rw submounts** for `30_Resources/Brands`, `30_Resources/Brainstorms`,
  `30_Resources/Flow`, and `10_Projects` (this tool writes under `10_Projects`). `~/.hermes` ↔ `/opt/data`;
  `.claude/skills` ↔ `/opt/skills` (ro). Tool roots: `HERMES_TOOL_ROOTS=/opt/skills:/opt/data/skills`.
- **Secrets live in `~/.hermes/.env`** (never committed): `CLAUDE_CODE_OAUTH_TOKEN` (the only required cred,
  from `claude setup-token`), `API_SERVER_KEY`, `CLAUDE_BRIDGE_TOKEN`, `MCP_OBSIDIAN_API_KEY` (optional),
  messaging tokens (optional). `scripts/doctor.sh` scans for plaintext secrets + 0.0.0.0 exposure.
- **`scripts/`:** `install.sh` (seed `~/.hermes`, gen tokens, build, up; `--wipe` tears down volumes),
  `doctor.sh` (read-only health/security), `claude-bridge.py` (**host-side**, token-gated `:8765`, runs
  `claude -p` on the host), `index-vault.py` (vault → Qdrant, Gemini embeddings), `seed-cron.sh`, bridge launchd service.
- **CI** (`.github/workflows/ci.yml`): dashboard job (`npm ci` → `typecheck` → `build`, Node 22); engine job
  (`compileall` syntax check — swap for pytest when ready). Triggers on push/PR to `main`.

## How changes are made here (the contract)

- **Conventional Commits** (`feat/fix/docs/test/chore`), strict TS in the dashboard, Python type hints in the
  engine, **files under ~500 lines**, validate input at boundaries. Add capability as a **skill** (`SKILL.md`
  [+ `tool.yaml`], validated against `engine/schemas/tool_manifest.schema.json`) discoverable at `/v1/tools` —
  don't patch the fork.
- **Local-first / private:** never bind `0.0.0.0` without TLS+auth; keep secrets out of git **and** the vault;
  run `scripts/doctor.sh` after any config/infra change.
- **Vault rules** (from the root vault `CLAUDE.md`): append/patch notes, **never delete or overwrite**; preserve
  `[[wikilinks]]`; honor PARA placement; one focused commit per edit.

## Landmines (read before editing)

1. **`dashboard-legacy-nextjs/` is dead** — all dashboard work is in `dashboard/` (Vite/React 19).
2. **Execution is Claude-only** now; ignore the 3-provider routing in `docs/architecture.md` for code decisions.
3. **Artifact reads must cache-bust** (`?_t=…`) — the run UIs poll on-disk JSON; stale otherwise.
4. **`API_SERVER_KEY` is required** — missing → engine API 401s and the dashboard proxy fails silently in dev
   (must `set -a; . ~/.hermes/.env; set +a` before `npm run dev`).
5. **The Claude bridge runs on the host, not in Docker** — `claude -p` needs host subscription/login.
6. **Don't bind ports to 0.0.0.0**; `doctor.sh` fails on it. Enabling a messaging platform exposes the agent —
   pairing + `*_ALLOWED_USERS` required; never `GATEWAY_ALLOW_ALL_USERS=true` on a reachable bot.
7. **Engine deps are exact-pinned on purpose** — don't loosen to ranges (supply-chain).
8. **Curator only touches `created_by: agent` skills and never deletes** — restore from `.archive/` if needed.
9. **This is a fork** of `NousResearch/hermes-agent` — upstream won't auto-merge; cherry-pick carefully.
